HTML

  • Elements that are nested inside other elements are called child elements. The element that holds all the children is known as the parent element.
  • The head element contains information about the webpage (metadata). Elements included within the head are not visible to the user.
  • The body element represents the visible content shown to the user.
  • The section elements are semantic elements that define a section in a document that contains thematic content.
  • The title element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO).
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The link element connects an external resource to the HTML document.
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource.

CSS

  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • A declaration contains the CSS property we want to apply and the value of the property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A class attribute allows a CSS rule to be shared to any element by assigning the rule to a class attribute with a class selector.
  • A class selector is designated by a period (.) preceding the class name.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: stages all files to be committed in the working branch
  • git commit -m "message": commits changes to repository with a short descriptive message.
  • git pull origin main: pulls in the main branch, syncing the local branch with the remote branch in GitHub.
  • git push origin branch-name: pushes up all local branch changes to the remote branch.

JavaScript

  • A variable (var) is a named container that allows us to store data in our code.
  • The var keyword declares a variable, followed by the uniquely named container.
  • The assignment operator = assigns the specific data stored in a variable, followed by the uniquely named data.
  • A string is a series of data contained within quotation marks, which is assigned to a variable. Numbers and Boolean values do not need quotation marks.
  • Each line of JavaScript code ends with a semicolon.
  • The word "console" in console.log refers to a test environment that developers use to check out their code.
  • A method, such as .log(), is a set of instructions that can be executed by a computer. The .log() method will output whatever we add inside the parentheses to the console.
  • Control flow is the order in which a computer executes code in a script. JavaScript has a top-down control flow.
  • Conditional statements and loops can alter the control flow.
  • The conditional statement "if" requires specified conditions to be met for the code to execute.
  • The conditional statement "else if" specifies alternate conditions under which the code will execute
  • A falsy value is 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false.
  • A truthy value is any other value, including a Boolean with the value of true.
  • The strict equality operator (===) checks to see if two values are equal, and returns a Boolean result true if the values are equal and false if the values are not equal.
  • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.
  • An index is a unique number that identifies the location of data within an array. In JavaScript, arrays are zero-indexed and sequential. The index is added to the end of an array name in square brackets (console.log(shapes[0]);).
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. By using a for loop, we can tell the computer to go through all of the data in our array and log them to the console until there is no data left. In order to use the for loop, we need the data's index and the array's length. The array length is important because the for loop needs to know how many times the code will run until there are no items left in the array. We can find the length of an array by using a built-in property of an array, appropriately named length.
  • A for loop contains the following three important statements: The first statement determines the starting point for our loop. Since we want to start at the beginning of the array, we declare a variable x and give it a value of 0 (because the first index of an array is always 0). The second statement is the condition. As long as the number in the variable x is less than the length of the array, the loop will keep running. The final statement is what allows the array to iterate over each item. Since indices follow a predictable pattern, we know the number increases by 1 each time. The code x++ is a shortcut way of writing x+1.
    • for(var x = 0; x < data.length; x++) {
      console.log(data[x]);
      }
